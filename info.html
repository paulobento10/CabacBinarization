Context-adaptive binary arithmetic coding (CABAC) é uma forma de codificação entropia usado nos padrões H.264 / MPEG-4 AVC e High Efficiency Video Coding (HEVC). É uma técnica de codificação sem perdas. O CABAC é notável por fornecer uma codificação muito melhor do que a maioria dos outros algoritmos de codificação de entropia usados na codificação de vídeo, e é um dos elementos-chave que fornece o esquema de codificação H.264 / AVC com melhor capacidade de codificação do que seus antecessores.

Por motivos de simplicidade resolvemos focar apenas e exclusivamente no processo de binarização do algoritmo cabac.

Explicação de binarização

No primeiro estado do codificador de entropia CABAC, o estado de binarização mapeia símbolos para uma sequência binária. A maioria dos elementos de sintaxe que devem ser codificados, representado por símbolos, alguns elementos de sintaxe são representados por um código binário. O codificador CABAC é um codificador binário,apenas as decisões binárias são codificadas. O processo binarizar um símbolo em um código binário é muito semelhante ao processo de conversão de um símbolo de dados em um código de comprimento variável.

A binarizacao usa 5 esquemas para codificar os elementos que recebe: Unary Code Tree, Exponential-Golomb coding, Truncated Unary, Unary/ th order Exp-Golomb, fixed length binarization.

Unary Code Tree

Este esquema ,baseia-se em atribuir o valor do elemento em 0's e terminar sempre com um valor "1" no fim da codificação

Exponential-Golomb coding

Para este algoritmo vamos codificar o valor de x+1

Conte os bits do valor, subtraia um e escreva esse número de zero bits iniciais precedendo a sequência de bits anterior.

Truncated Unary

Para um exemplo de 10 elementos(0-9), é necessário 4 bits para codificar em binário, mas existe 2⁴-10=6 codigos binários não usados.

Então valores de entrada menores que 6 têm o primeiro bit descartado, enquanto valores de entrada maiores ou iguais a 6 são compensados por 6 até o final do espaço binário.

Concatenation of the first and second scheme (UEGK)

É utilizado para binarização de Simbolos de valor absoluto de nível de coeficiente residual e MVD. Gera prefixo da string bin, e EGk é adotado para gerar o sufixo com k definido para 0 e 3 para o nível de coeficiente e MVD, respectivamente. A TU é simples e permite adaptação rápida de probabilidade de codificação símbolo. No entanto, é apenas benéfico para valores pequenos. Para grandes valores, a cadeia de sufixos gerada pelo programa EGk oferece um bom ajuste à distribuição de probabilidades, e também é utilizada a codificação da caixa de bypass para reduzir a complexidade de computação.

Fixed length

A codificação de comprimento fixo baseia-se na seguinte forma, codificar o elemento recebi no valor binário com o comprimento de o valor maximo, ou seja , para o seguinte :

codificar 100,255,2 : como 255 é o maior valor em binário (11111111) , necessita de 2³ bits para ser codificado , logo quando codificarmos o valor 2 será como 00000010.
